# Create visual pull-requests

Visual pull-requests in Widgetbook Cloud are automatically created whenever a Widgetbook project is connected to a Git provider's repository based on their code-based pull-requests.

## Requirements

To use visual pull-requests
- [connect your Widgetbook Cloud project with the repository of your Git provider](/cloud/projects/connect#connect-project)
- successfully [upload builds](/cloud/builds/upload#upload-build-using-cicd) for the base and head commits of the pull-request

## How visual pull-requests connect with builds

Pull-requests in your Git provider as well as in Widgetbook operate on a base and head commit. 
The base commit is the commit that the pull-request is based on, and the head commit is the commit that contains the changes in the pull-request.

<Image src="/assets/cloud/visual-pull-request/pr-visualisation-light.png" theme="light" caption="PR #1 aims to merge head commit z into base commit a by creating merge commit m" />
<Image src="/assets/cloud/visual-pull-request/pr-visualisation-dark.png" theme="dark" caption="PR #1 aims to merge head commit z into base commit a by creating merge commit m" />

To calculate the visual changes in a pull-request, Widgetbook Cloud needs to have snapshots for both the base and head commits of the pull-request.
Those snapshots are provided by [uploading builds](/cloud/builds/upload#upload-build-using-cicd) to Widgetbook Cloud.

## Missing builds

If a visual pull-request misses builds, Widgetbook Cloud will show a warning in the pull-request overview.

<Image src="/assets/cloud/visual-pull-request/missing-build" caption="The pull-request is missing builds for both the base and head commits." />

Use the instructions about [uploading builds](/cloud/builds/upload#upload-build-using-cicd) to ensure that all necessary builds are available.

<Warning>
Caution for uploading the head build of a visual pull-request:
Some Git providers like GitHub run their CI/CD pipeline on merge commit `m` when the pipeline is triggered by a pull request.
This will cause Widgetbook Cloud to still miss the head commits `z` for the pull-request and can cause confusion while setting up visual pull-requests.
</Warning>

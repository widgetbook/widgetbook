# Styling Documentation

By default, DocBlocks use Widgetbook's built-in theme. This works well out of the box, but you might want your documentation to match your app's design system instead.

There are two approaches:

1. **Override Widgetbook's UI theme** ŌĆö quick, limited to what `ThemeData` exposes.
2. **Build custom DocBlocks with your own styling** ŌĆö full control, uses your app's theme.

## Override Widgetbook's UI Theme

Widgetbook's built-in DocBlocks read their text styles and colors from `Theme.of(context)`. You can override these via `Config`:

```dart
final config = Config(
  components: components,
  lightTheme: ThemeData(
    colorScheme: ColorScheme.fromSeed(seedColor: Colors.indigo),
    textTheme: const TextTheme(
      headlineLarge: TextStyle(fontFamily: 'Poppins', fontSize: 32),
      headlineMedium: TextStyle(fontFamily: 'Poppins', fontSize: 24),
      bodyMedium: TextStyle(fontSize: 16, height: 1.6),
    ),
  ),
);
```

This changes how all built-in DocBlocks render ŌĆö `TitleDocBlock` uses `headlineLarge`, `SubtitleDocBlock` uses `headlineMedium`, and `TextDocBlock` uses `bodyMedium`.

See [Widgetbook Theme](/configure/widgetbook-theme) for all available parameters.

For most teams, this won't be enough. The built-in DocBlocks only expose a limited set of styles. For full control, build your own.

## Build Custom-Styled DocBlocks

DocBlocks are Flutter widgets. Inside `build(context)`, you have full access to the widget tree ŌĆö which means you can style them however you want.

### Using Your App Theme

If you use the [Theme Addon](/addons/theme-addon), your app's theme is already available in the widget tree. Custom DocBlocks can read it the same way your app widgets do:

```dart
// With MaterialThemeAddon
class BrandedTitleDocBlock extends DocBlock {
  const BrandedTitleDocBlock({super.key});

  @override
  Widget build(BuildContext context) {
    final state = WidgetbookState.of(context);
    final appTheme = Theme.of(context); // Your app's ThemeData

    return Text(
      state.component!.name,
      style: appTheme.textTheme.displayMedium,
    );
  }
}
```

```dart
// With a custom ThemeAddon<AppThemeData>
class BrandedTitleDocBlock extends DocBlock {
  const BrandedTitleDocBlock({super.key});

  @override
  Widget build(BuildContext context) {
    final state = WidgetbookState.of(context);
    final appTheme = AppTheme.of(context); // Your custom theme

    return Text(
      state.component!.name,
      style: TextStyle(
        fontSize: 32,
        color: appTheme.primaryColor,
      ),
    );
  }
}
```

This way, your documentation stays in sync with your app's design tokens ŌĆö if you change a color or font in your theme, the docs update automatically.

### Hardcoded Styles

You don't have to use any theme at all. Style your DocBlocks directly:

```dart
class CalloutDocBlock extends DocBlock {
  const CalloutDocBlock(this.text, {super.key});

  final String text;

  @override
  Widget build(BuildContext context) {
    return Container(
      padding: const EdgeInsets.all(16),
      decoration: BoxDecoration(
        color: const Color(0xFFF0F4FF),
        borderRadius: BorderRadius.circular(8),
        border: Border.all(color: const Color(0xFF3B82F6)),
      ),
      child: Text(
        text,
        style: const TextStyle(fontSize: 14, color: Color(0xFF1E3A5F)),
      ),
    );
  }
}
```

### Wire It Up

Use your custom blocks via `docsBuilder` ŌĆö globally or per component:

```dart
final config = Config(
  components: components,
  docsBuilder: () => [
    const BrandedTitleDocBlock(),
    const DartCommentDocBlock(),
    const StoriesDocBlock(),
  ],
  addons: [
    MaterialThemeAddon(themes: { /* ... */ }),
  ],
);
```

See [Customize](/docs/customize) for per-component overrides and list manipulation helpers like `insertAfter`, `insertBefore`, and `replaceFirst`.

## Use Built-in DocBlocks as Reference

The built-in DocBlocks are intentionally simple ŌĆö most are under 20 lines. Browse the source to see how they access component data and render content:

[View DocBlock source on GitHub](https://github.com/widgetbook/widgetbook/tree/main/packages/widgetbook/lib/src/core/docs)

Key patterns you'll find:

- **`WidgetbookState.of(context)`** ŌĆö access the current component, its stories, doc comments, and args.
- **`state.component!.name`** ŌĆö the component name (used by `ComponentNameDocBlock`).
- **`state.component!.docComment`** ŌĆö the Dart doc comment extracted by build_runner (used by `DartCommentDocBlock`).
- **`state.component!.stories`** ŌĆö the list of stories with their args (used by `StoriesDocBlock`).
